# Tauri Setup and Security

## Install

```sh
pnpm add @codenhub/tauri-plugin-webview @tauri-apps/api
```

The frontend package is ESM and must run in a Tauri v2 WebView. Browser-only,
SSR, and Node.js execution are not supported runtime contexts.

## Register the Rust Commands

Copy `node_modules/@codenhub/tauri-plugin-webview/rust/webview_commands.rs` to
the consumer's Rust source tree, for example
`src-tauri/src/webview_commands.rs`. Register both commands:

```rust
mod webview_commands;

tauri::Builder::default()
    .invoke_handler(tauri::generate_handler![
        webview_commands::navigate_webview,
        webview_commands::reload_webview,
    ]);
```

The source uses the consumer's existing Tauri v2 Rust dependency. It adds
`navigate_webview` and `reload_webview` application commands; it is not a Cargo
plugin and requires no plugin registration.

By default, Tauri permits registered application commands to application
windows and webviews. If the app restricts application commands through its
build manifest, allow these two commands there as well.

## Grant Core Permissions

Add permissions to a capability assigned to each bundled caller. A capability
using the entire public surface needs the following core permissions:

```json
{
  "permissions": [
    "core:webview:allow-create-webview-window",
    "core:webview:allow-set-webview-zoom",
    "core:window:default",
    "core:window:allow-destroy",
    "core:window:allow-hide",
    "core:window:allow-set-focus",
    "core:window:allow-set-position",
    "core:window:allow-set-size",
    "core:window:allow-show"
  ]
}
```

Tauri's `WebviewWindow` deliberately uses window commands for overlapping APIs,
and `core:window:default` supplies enumeration used by lookup and listing. The
created and error events used during creation are local to `WebviewWindow` and
do not need a core event permission. Remove operation permissions for APIs the
caller does not use. Confirm identifiers against the schemas generated by the
exact Tauri version in the consumer app.

## Trust Boundaries

Only grant creation and control permissions to trusted bundled webviews. Tauri
capabilities are assigned by window/webview label, and overlapping capabilities
merge their authority.

The Rust navigation command parses the supplied URL and navigates without an
origin allowlist. Treat every URL passed to `spawnWebview` or `navigate` as
security-sensitive, enforce an application-specific allowlist before calling,
and configure Tauri's remote content and Content Security Policy separately.
Do not grant remote pages Tauri API access merely because this package displays
them.
